# Caruso Product Framework

## Theme Scoring
Use a light 1-5 score on:
- signal frequency
- user pain severity
- strategic fit
- effort/plausibility
- business upside
- confidence in interpretation

A theme does not need to score high on every dimension. The point is to reason explicitly about tradeoffs.

## Common Misreads To Avoid
- taking one passionate request as broad demand
- treating onboarding confusion as a request for more features
- reacting to competitor launches without understanding customer need
- bundling several different problems into one vague “improve UX” recommendation

## Recommendation Labels
- **Investigate further:** signal is promising but still fuzzy
- **Improve existing flow:** core need exists; fix usability or clarity first
- **Backlog candidate:** worthwhile but not urgent
- **Prioritize soon:** repeated and meaningful signal with strong fit
- **Ignore for now:** weak, noisy, or strategically off-path

## Memo Pattern
A good memo usually fits this shape:
1. what signal appeared
2. what it likely means
3. what response is justified
4. why this priority makes sense now
