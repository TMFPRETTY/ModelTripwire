# Support Triage Patterns

## Quick Classification Examples

### Urgent
- account locked and customer cannot work
- suspicious login or compromised account concern
- production-breaking failure
- deadline today and user is blocked

### High
- repeated failure without workaround
- customer upset and considering churn
- billing problem needing review soon

### Normal
- how-to question
- feature clarification
- minor bug report with workaround
- request for status update

### Low
- vague suggestion
- duplicate note already handled elsewhere
- non-actionable commentary

## Summary Pattern
Good summary formula:
- who wrote
- what went wrong or what they want
- what impact it has
- what timing/risk exists

## Safe Draft Pattern
Use a simple shape:
1. acknowledge
2. restate issue briefly
3. say what happens next
4. provide expectation if known

Example:
"Thanks for flagging this. I understand you’re running into [issue], which is affecting [impact]. We’re reviewing it now and will follow up with the next step shortly."
